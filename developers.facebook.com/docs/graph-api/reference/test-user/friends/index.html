Graph API Version
v2.12

/{test-user-id}/friends

The friends of a test user. This is identical to the /{user-id}/friends edge aside from the publishing operation explained below.

Related Guides

  • Managing Test Accounts using the App Dashboard

Publishing

By publishing using this edge, you send friend requests between test users, and confirm those friend requests:

POST /v2.12/{test-user-1}/friends/{test-user-2} HTTP/1.1
Host: graph.facebook.com
/* PHP SDK v5.0.0 */
/* make the API call */
try {
  // Returns a `Facebook\FacebookResponse` object
  $response = $fb->post(
    '/{test-user-1}/friends/{test-user-2}',
    array (),
    '{access-token}'
  );
} catch(Facebook\Exceptions\FacebookResponseException $e) {
  echo 'Graph returned an error: ' . $e->getMessage();
  exit;
} catch(Facebook\Exceptions\FacebookSDKException $e) {
  echo 'Facebook SDK returned an error: ' . $e->getMessage();
  exit;
}
$graphNode = $response->getGraphNode();
/* handle the result */
/* make the API call */
FB.api(
    "/{test-user-1}/friends/{test-user-2}",
    "POST",
    function (response) {
      if (response && !response.error) {
        /* handle the result */
      }
    }
);
/* make the API call */
new GraphRequest(
    AccessToken.getCurrentAccessToken(),
    "/{test-user-1}/friends/{test-user-2}",
    null,
    HttpMethod.POST,
    new GraphRequest.Callback() {
        public void onCompleted(GraphResponse response) {
            /* handle the result */
        }
    }
).executeAsync();

// with `FBSDKShareOpenGraphContent`

}];

Permissions

  • The user access token a test user is required to send any friend requests from them, or to confirm any friend requests send to them.
  • Only the recipient of a friend request can confirm that request.
  • This call will only work with Test users associated with a Facebook app.

As an access token for both test accounts is required, you can't friend two test users via the Graph API if one or both of them have not installed the app. Instead you can add them as friends of each other using the Test Users section of the app dashboard

Fields

This call does not have any parameters, the second test user must be used in the API path itself.

Response

If successful:

{
  "success": true
}

Otherwise a relevant error message will be returned.
