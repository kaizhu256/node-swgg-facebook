Permissions

  • Any valid access token if the photos are public.
  • A user access token user_photos permission to retrieve any photos that the session user has uploaded that are not public.

Fields

A list of Photo objects representing the photos in the album.

Publishing

There are two separate ways to publish photos to Facebook:

  1. Capture a photo via file upload as multipart/form-data then use the source parameter:

POST /v2.12/{album-id}/photos HTTP/1.1
Host: graph.facebook.com
source=%7Bimage-data%7D

    '{access-token}'
  );
} catch(Facebook\Exceptions\FacebookResponseException $e) {
  echo 'Graph returned an error: ' . $e->getMessage();

Permissions

  • A user access token with publish_actions permission can be used to publish new photos.

Fields

Name Description Type

source

The photo, encoded as form data. Either this or url field is required, but both should not be used together.

multipart/form-data

url

The URL of a photo that is already uploaded to the internet. Either this or source is required, but both should not be used together.

string

message

The description of the photo, used as the accompanying status message in any feed story.

string

place

Page ID of a place associated with the Photo.

string

no_story

If set to true, this will suppress the feed story that is automatically generated on a person's profile when they upload a photo using your app.

boolean

Response

If successful, you will receive a response with the following information. In addition, this endpoint supports read-after-write and can immediately return any fields returned by read operations.

Name Description Type

id

The newly created photo ID

string

post_id

The ID of the post story about the uploaded photo.

string

Deleting

You can't delete using this edge, however you can delete each photo using the /{photo-id} node.
